# 📅 April 26 – Cyber Security Learning Log

## 🧠 Introduction to Behavioral Analytics
**Behavioral analytics** in cybersecurity focuses on studying and analyzing user and system behavior to detect anomalies and potential threats.

### 🔹 Purpose:
- Understand normal behavior patterns
- Detect unusual or malicious activities
- Improve overall security monitoring

---

## 📊 Identifying User Behavior Patterns

### 🔹 What It Involves:
- Tracking login times and locations
- Monitoring application usage
- Observing data access patterns
- Analyzing frequency of actions

### 🔹 Example:
- A user normally logs in from one location but suddenly logs in from another country → flagged as unusual behavior

---

## 🚨 Detecting Suspicious Activities

### 🔹 Indicators:
- Multiple failed login attempts
- Accessing sensitive data without permission
- Unusual data transfers
- Sudden increase in system activity

### 🔹 Methods:
- Real-time monitoring
- Alert systems
- Anomaly detection techniques

---

## 🤖 Using AI and Machine Learning in Security Analytics

### 🔹 How It Works:
- Learns normal behavior patterns over time
- Identifies deviations automatically
- Uses predictive analysis to detect threats early

### 🔹 Advantages:
- Faster threat detection
- Handles large volumes of data
- Detects unknown (zero-day) attacks
- Reduces false positives

---

## 🔧 Applications:
- User and Entity Behavior Analytics (UEBA)
- Fraud detection systems
- Intrusion detection systems

---

## ✅ Summary
- Behavioral analytics helps understand and monitor user activity
- Identifying normal patterns is key to detecting anomalies
- Suspicious activities can be flagged through monitoring and analysis
- AI and machine learning enhance detection accuracy and efficiency
