# 📅 April 25 – Cyber Security Learning Log

## 🧩 Introduction to Application Security
**Application security** focuses on protecting software applications from threats and vulnerabilities throughout their lifecycle.

### 🔹 Objectives:
- Prevent unauthorized access
- Protect sensitive data
- Ensure secure coding practices
- Reduce vulnerabilities in applications

### 🔹 Common Threats:
- SQL Injection
- Cross-Site Scripting (XSS)
- Broken authentication
- Data breaches

---

## 🔐 Understanding AAA Framework
The **AAA Framework** is a security model used to control access and track user activities.

---

### 🔑 Authentication
**Authentication** verifies the identity of a user or system.

#### 🔹 Methods:
- Username and password
- OTP (One-Time Password)
- Biometrics (fingerprint, face recognition)

---

### 🚪 Authorization
**Authorization** determines what an authenticated user is allowed to do.

#### 🔹 Examples:
- Access to files or databases
- Permission levels (admin, user, guest)

---

### 📊 Accounting
**Accounting** tracks user activities and resource usage.

#### 🔹 Functions:
- Logs user actions
- Monitors system usage
- Helps in auditing and compliance

---

## 🔐 Role of Encryption in Securing Applications

### 🔹 Importance:
- Protects data from unauthorized access
- Ensures confidentiality and integrity
- Secures data during transmission and storage

### 🔹 Types:
- **Data in Transit** – Secured using HTTPS, TLS
- **Data at Rest** – Encrypted storage (databases, disks)

### 🔹 Benefits:
- Prevents data breaches
- Protects user credentials
- Builds trust in applications

---

## ✅ Summary
- Application security protects software from cyber threats
- AAA framework ensures secure access control and monitoring
- Encryption safeguards sensitive data in applications
